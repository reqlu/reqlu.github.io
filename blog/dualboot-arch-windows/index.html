Notice AMLES Installation de postes de travail en Dual Boot (VM ProxMox) Part1.


Dans le cadre d’une SAE en BUT Informatique
Par Lucas R. & Maxime L.

I. Partitionnement du disque.

A. Installation de l’image ArchLinux :

  • Démarrez votre vm et appuyez sur la touche <ESC> pour pouvoir accéder à la fenêtre du BIOS.
  • Allez dans Boot Manager.
  • Sélectionnez le CD contenant l’image iso pour l’installation de Archlinux. Ici on a : UEFI QEMU DVD-ROM QM00001.
  • Sélectionnez le modèle d’installation : Arch Linux install medium (x86_64, UEFI).
  • Une fois que le shell apparaît, vous pouvez écrire vos commandes.

B. Quelques réglages et partitionnement du disque :


  • Pour commencer, réglez votre clavier en AZERTY car il est configuré par défaut en
    QWERTY. Pour cela il faut, écrire la commande suivante: loadkeys fr.
  • Utiliser la commande fdisk -l pour lister les disques présent ou la commande lsblk
    Remarque : fdisk -l permet d’afficher la taille des secteurs et aussi la position dans le
    système de fichier    .
  • Trouvez un disque ayant suffisamment d’espace alloué, ex: le disque vda qui a une
    capacité de 40Go.
  • Pour partitionner les disques, il suffit de faire la commande cgdisk avec le chemin du
    disque     : cgdisk /dev/vda.

Remarque : si un message de prévention apparaît, le lire puis appuyer sur la touche entrée.

  • Une fenêtre de dialogue va apparaître pour de créer les partitions

  • Sélectionnez NEW.

  • Définissez le premier secteur, pour cela appuyer sur la touche <ENTER> pour donner la valeur par défaut.

  • Définissez la taille du secteur, pour cela il faut comprendre que la taille d’un secteur vaut 512
    octet     donc si l’on souhaite une partition de 512Mo, vous devrez allouer 1 000 000 de secteurs.
    Pour calculer la taille total il vous faut convertir la taille de ce qu l’on veut en octet donc 512Mo
    = 512 1024 * 1024 = 536 870 912 octets puis on divise par la taille d’un secteur donc 512, on obtient 1 048 576.

  • Ensuite, on vous demande le code du type de la partition, vous pouvez rentrer la touche <L>
    pour chercher le code de la partition demandé.

  • Vous pouvez écrire du texte pour une recherche
    plus rapide, exemple : EFI permet de trouver son code qui correspond à ef00.

  • Rentrez le code correspondant.

  • Donnez un nom à la partition (ex: EFI).

Répétez ses actions jusqu’à avoir crée toutes les partitions

Remarque : attention vous devez séléctionner l’espace libre dans lequelle vous voulez écrire.
Aide : le code associé à Linux est 8300

  • Vérifier l’intégrité des données du disque en séléctionnant VERIFY.
  • Sélectionnez WRITE pour sauvegarder les partitions.
  • Appuyez sur le bouton QUIT.
  • Utilisez la commande reboot pour redémarrer votre machine et appuyer sur la touche <ESC> pour accéder au BIOS.

II. Installation de Windows sur l’espace restant libre.

A. Installation de l’image de Windows :


  • Une fois dans le Boot Manager, sélectionnez le CD contenant l’ISO Windows 7 (UEFI QEMU DVD-ROM QM00003) et appuyez sur une touche pour valider.

B. Installation de Windows 7 :

  • Sélectionnez le clavier en français et la langue en français et appuyer sur le bouton
    installer.
  • Lisez et acceptez les termes du contrat de licence.
  • Sélectionnez l’option personnalisé
  • Vous devez charger un pilote et appuyer sur parcourir.
  • Accédez à l’arborescence du CD virtio et accédez au dossier viostor puis w7 puis
    adm64 et appuyez sur OK ( E:\viostor\w7\amd64\viostor.inf).
  • Cliquez sur le bouton Suivant en ayant le pilote.
  • Sélectionnez l’espace non alloué sur le disque qui équivaut à 24.5Go, c’est le disque0.
  • L’ordinateur effectue un démarrage afin d’effectuer la fin de l’installation.
  • Entrez le nom d’utilisateur de votre choix ainsi que le nom de l’ordinateur et cliquez sur
    sélectionnez.
  • Définissez le mot de passe de votre machine.
  • Utilisez seulement les paramètres recommandés.
  • Choisissez le bon fuseau horaire (Paris) et la bonne heure, date.
  • Laissez l’ordinateur redémarrer.

C. Paramétrage du réseau :

  • Une fois sur votre bureau : configurez le réseau, en allant dans : gestionnaire de l’ordinateur -> Outils système -> gestionnaire de périphériques.
  • Trouvez dans l’arborescence le périphérique correspondant au “contrôleur Ethernet”.
  • Faites clique droit sur le “contrôleur Ethernet” et sélectionnez “mettre à jour le pilote”.
  • Indiquez le lecteur correspondant au CD virtio.

III. Installation du système ArchLinux.



Utile: https://wiki.archlinux.org/title/Installation_guide_(Français)

Après en avoir fini avec l’installation du système d’exploitation windows7, on redémarre
la machine afin d’accéder à la console linux afin de formater nos disques.

Pour ce faire, on utilise la commande lsblk afin d’afficher les différentes partitions du disque. cette commande va nous permettre de savoir où se trouve le linux, et le linux home.

On formate donc /dev/vda2 et /dev/vda3 (10G et 5G). On n’oublie pas que le format est ext4 ! car ext4 est le système de fichiers principalement destiné aux systèmes basés sur
Linux.

Pour formater on utilise la commande mkfs.ext4 ce qui donne :

root@archiso ~ # mkfs.ext4 /dev/vda2
root@archiso ~ # mkfs.ext4 /dev/vda3


Une fois les commandes exécutées, on monte /mnt sur le disque /dev/vda2. Pour ce faire on utilise la commande :

mount /dev/vda2 /mnt

et on vérifie le bon fonctionnement de la commande avec lsblk.


Une fois la vérification effectuée, on créé deux dossiers dans /mnt portant le nom de boot
et home avec la commande :

root@archiso /mnt # mkdir boot && mkdir home

Pour la suite, nous allons avoir besoin de l’éditeur NANO, on l’installe à l’aide de la commande :

pacman -S nano.

Ensuite, nous allons devoir chercher les serveurs mirror de arch linux afin d’obtenir les dépôts à jour. Avant de configurer les mirror, ont test la connexion de la machine à l’aide de la commande:

ping google.fr


Une fois le test de la connexion validé, on génère le fichier /etc/pacman.d/mirrorlist.
Ce fichier sera la liste des mirror français. Pour cela, on utilise la commande :

reflector --country France --age 12 --protocol https --sort rate --save /etc/pacman.d/mirrorlist.

Après ça, on installe les paquets essentiels au système avec la commande:

pacstrap -K /mnt base linux linux-firmware.

Une fois ceci fait, on ne doit garder que deux serveurs parmi cette liste.
À l’aide du site : https://archlinux.org/mirrorlist/

Nous allons génèrer une liste de serveurs actifs, avec les bons paramètres et éditer à l’aide du package nano le fichier /etc/pacman.d/mirrorlist afin de garder que deux serveurs qui sont communs aux deux listes.

Ne pas oublier d’enlever les # ! et CTRL+X et Y pour sauvegarder !


Nous pouvons désormais passer à la configuration du système. On commence par monter notre Linux Home sur le /mnt/home et par la suite, une génération du fichier fstab se fera dans /mnt/etc/fstab.

(le fichier fstab est la table des différents systèmes de fichiers sur un ordinateur sous Unix/Linux : il contient une liste des disques utilisés au démarrage et des partitions de ces disques.)

On utilise la commande :

genfstab -U /mnt >> /mnt/etc/fstab.

Une fois la commande terminée, on “s’emprisonne” dans arch-chroot avec la commande :

arch-chroot /mnt

(On chroot dans le nouveau système, ce qui veut dire que notre /mnt devient notre répertoire courant.)
Après ça, on définit notre fuseau horaire avec la commande :

ln -sf /usr/share/zoneinfo/Europe/Paris /etc/localtime
(Fuseau Horaire Paris)

On modifie le fichier /etc/locale.gen afin de supprimer le # devant le fr_FR.UTF-8 , pour passer d’un commentaire à une commande.
On définit la langue de la machine et le clavier avec la commande :

echo LANG='fr_FR.UTF-8' > /etc/locale.conf && echo KEYMAP=fr > /etc/vconsole.conf

Désormais, on définit également notre fichier hosts, qui se trouve dans /etc/hosts et on place dedans :

127.0.0.1 localhost
::1
localhost
127.0.1.1 nomdevotremachine

à l’aide de la commande :

nano /etc/hosts.

On place aussi notre nomdemachine dans /etc/hostname avec la commande :

echo "nomdevotremachine" > /etc/hostname


Ensuite, on installe les paquets concernant le réseau tel que le paquet dhcpcd , nettcl, ifplugd avec la commande :

pacman -S netctl dhcpcd ifplugd.

IV. Installation du dual boot

Après avoir fini de configurer Arch Linux, nous devons désormais mettre en place le dualboot, qui nous permettra par la suite de pouvoir choisir au démarrage de la machine si nous voulons aller sur Windows7 ou ArchLinux en fonction de nos besoins.
Pour commencer, tout en restant dans le arch-chroot, nous allons créer un répertoire dans / nommé “esp” à l’aide de la commande :

cd / && mkdir esp.

Ce répertoire ESP (EFI System Partition) va donc servir de lieu de stockage pour les chargeurs d’amorçage EFI, (bootloader) qui est un “logiciel” permettant delancer un ou plusieurs systèmes d’exploitation. Ensuite, dans /esp on monte l’EFI qui se trouve dans /dev/vda1 à l’aide de la commande :

mount /dev/vda1 /esp

Avant d’installer et de lancer le service GRUB nous devons vérifier notre architecture
(x86-64). Pour ce faire on utilise la commande :
uname -m

Nous pouvons désormais installer le service GRUB avec la commande :

pacman -Syu grub efibootmgr

Une fois le paquet installé, on peut lancer ce service en précisant le répertoire où l’on désire l’installer. Qui sera donc dans /esp. On utilise la commande :

grub-install --target=x86_64-efi --boot directory=/esp –efi-directory=/esp --bootloader id=arch_grub.

Une fois la commande effectuée, on installe le paquet os-prober qui va servir à rechercher les différents systèmes d’exploitation installés sur la machine. On utilise la commande :

pacman -S os-prober.

Ensuite, nous allons désactiver os-prober depuis le fichier /etc/default/grub. Nous devons retirer le # devant la ligne :

GRUB_DISABLE_OS_PROBER=false.

Avec la commande :

nano /etc/default/grub


À présent, nous pouvons créer le fichier de configuration du GRUB à l’aide de la commande :

grub-mkconfig -o /esp/grub/grub.cfg.

La commande étant terminée on garde une copie du fichier original bootx64.efi sous le nom bootx64.efi.initial dans le même répertoire. Le fichier utilise Boot/bootx64.efi On doit donc faire en sorte que son contenu soit remplacé par celui de grubx64.efi avec la commande :

mv /esp/EFI/Boot/bootx64.efi /esp/EFI/Boot/bootx64.efi.initial && cp /esp/EFI/arch_grub/grubx64.efi /esp/EFI/Boot/bootx64.efi

Nous pouvons à présent définir un mot de passe pour le superutilisateur (root) avec la
commande : passwd.

Par défaut le mot de passe sera azerty. Suite à ça, on démonte nos disques et on reboot avec la commande :

umount /mnt/home && umount /mnt/esp && umount /mnt && reboot

Une fois redémarrée, une dernière chose est à faire ! Le service doit DHCP doit etre automatiquement lancée à chaque démarrage. On utilise la commande :

systemctl enable dhcpcd.
